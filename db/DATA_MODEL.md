## Conventions

These apply uniformly across all 14 tables in the schema. They were decided during Phase 5 before any table design and are not repeated in individual table sections.

### Naming

**PostgreSQL side (snake_case):**

- Tables: plural, `snake_case` — `properties`, `property_access`, `account_numbers`.
- Columns: `snake_case` — `created_at`, `owner_id`, `valid_from`.
- Indexes: `<table>_<purpose>_idx` — `properties_owner_id_idx`.
- Constraints: descriptive names — `bills_amount_check`, `contracts_no_overlap_excl`.

**TypeScript / Drizzle side (camelCase):**

- Drizzle table variable: same plural form as DB table name — `pgTable('properties')` assigned to `properties`.
- Columns: `camelCase` in TypeScript, mapped to `snake_case` in the DB via Drizzle's column constructor second argument.
- Domain types: singular — `type Property = typeof properties.$inferSelect`.
- Auth.js OAuth token fields are an exception from camelCase only in the DB layer (`refresh_token`, `access_token`); TS layer still uses `refreshToken`, `accessToken`.

### Primary Keys

All primary keys are **UUID v4**, generated by PostgreSQL's `gen_random_uuid()`.

**Why UUID over auto-increment:**

- Multi-tenant safety — no information leaked through sequential IDs.
- Distributed-friendly — IDs can be generated before DB insert if needed.
- Branded types in TypeScript enable strict type discrimination: `PropertyId`, `UserId` are structurally distinct despite both being strings.

In Drizzle:

```ts
id: uuid("id").primaryKey().defaultRandom().$type<PropertyId>();
```

### Timestamps

All tables (except Auth.js service tables where specified) carry:

| Column      | Type                        | Null | Default |
| ----------- | --------------------------- | ---- | ------- |
| `createdAt` | `timestamptz` (TIMESTAMPTZ) | NO   | `now()` |
| `updatedAt` | `timestamptz`               | NO   | `now()` |
| `deletedAt` | `timestamptz`               | YES  | —       |

- All timestamps use `withTimezone: true` (PostgreSQL `TIMESTAMPTZ`). Storage is in UTC, rendering is in the user's timezone.
- `updatedAt` is auto-maintained at application level via Drizzle's `.$onUpdate(() => new Date())`.
- `deletedAt IS NULL` means active. Non-null means soft-deleted at that moment.

### Nullability

**Default: NOT NULL.** Nullability is an opt-in choice, documented per column. Reasons a column may be nullable:

- The field represents "absence of value" with specific meaning (e.g., `validTo = NULL` means "still valid").
- The field is genuinely optional from the user's perspective (e.g., `notes`).
- The field is populated lazily or only in specific contexts.

### Foreign Keys and Cascade Behavior

| Scenario                                     | Policy               |
| -------------------------------------------- | -------------------- |
| Child → parent (main owning relationship)    | `ON DELETE CASCADE`  |
| Child → reference/lookup table               | `ON DELETE RESTRICT` |
| Optional child → parent where child survives | `ON DELETE SET NULL` |

**Interaction with soft delete:** physical cascade matters only for hard delete (admin-only). Soft delete cascades are handled at application level — a dedicated service function (`softDeleteProperty`) updates `deletedAt` on all related rows in one transaction.

### Check Constraints

Business invariants expressible at the row level are enforced via CHECK constraints in the database — defense in depth. Examples: `amount >= 0`, `zone_count IN (1, 2, 3)`, `valid_to > valid_from`.

Cross-table or cross-row invariants (e.g., "at least one owner per property") live at the application level.

### Enums

Text columns + CHECK constraint, not native PostgreSQL `ENUM` types.

Rationale: easier to evolve (changing allowed values doesn't require an `ALTER TYPE` migration), cleaner Drizzle integration, natural mapping to TypeScript string unions.

In Drizzle:

```ts
systemRole: text("system_role", { enum: ["user", "admin"] })
  .notNull()
  .default("user");
```

With a CHECK constraint at the DB level: `CHECK (system_role IN ('user', 'admin'))`.

### Temporal Data

For entities with history (contracts, tariffs, account numbers, payment details, meters):

- `validFrom TIMESTAMPTZ NOT NULL` — period start (inclusive).
- `validTo TIMESTAMPTZ NULL` — period end (exclusive). NULL = still valid.
- Half-open interval semantics `[validFrom, validTo)`.
- Exclusion constraints prevent overlapping active periods within a logical group (e.g., no two active contracts for the same service).
- `CHECK (valid_to IS NULL OR valid_to > valid_from)`.

## Entity Relationship Overview

```
                    User
                     │
                     │ systemRole: 'user' | 'admin'
                     │
        ┌────────────┴───────────────┐
        │                            │
        │ (owner via PropertyAccess) │ (owner of)
        ▼                            ▼
    Property ──PropertyAccess──> User     Provider
        │       (role)                        │
        │                                     │
        ├─► Service ──► Contract ─────────────┘ (references)
        │    (type)       │
        │                 ├─► Tariff           (temporal, many)
        │                 ├─► AccountNumber    (temporal, many)
        │                 └─► PaymentDetails   (temporal, many)
        │
        └─► Meter ──► Reading
             (serviceType)

    Service ──► Bill  (period attribution)
    Service ──► Payment
```

Key relationships:

- **User ↔ Property** is many-to-many via `property_access` with roles.
- **Service** is the hub of the CRM domain — it links a Property to a ServiceType, and owns Contracts, Bills, Payments.
- **Contract** is the aggregate that groups Provider + temporal Tariffs/AccountNumbers/PaymentDetails.
- **Meter** belongs to Property (physical device) but is linked to ServiceType for semantic matching with Services.
- **Reading** is owned by Meter. Not directly tied to Service.
- **Bill** and **Payment** are independent ledger entries. No FK between them.

## Block 1: Identity & Access

### `users`

Application users.

| Column          | Type                                        | Null | Default             | Notes                                                 |
| --------------- | ------------------------------------------- | ---- | ------------------- | ----------------------------------------------------- |
| `id`            | `uuid`                                      | NO   | `gen_random_uuid()` | Branded `UserId`.                                     |
| `name`          | `text`                                      | YES  | —                   | From Google profile or manual.                        |
| `email`         | `text`                                      | NO   | —                   | Required by Auth.js.                                  |
| `emailVerified` | `timestamptz`                               | YES  | —                   | Required by Auth.js. NULL if never verified.          |
| `image`         | `text`                                      | YES  | —                   | Avatar URL.                                           |
| `systemRole`    | `text` enum `'user' \| 'admin'`             | NO   | `'user'`            | Application-level role. Admin via `ADMIN_EMAILS` env. |
| `locale`        | `text` enum `'en' \| 'uk' \| 'ru'`          | NO   | `'en'`              | UI language.                                          |
| `theme`         | `text` enum `'light' \| 'dark' \| 'system'` | NO   | `'system'`          | UI theme preference.                                  |
| `timezone`      | `text`                                      | NO   | `'Europe/Kyiv'`     | IANA timezone. Hardcoded default in MVP.              |
| `createdAt`     | `timestamptz`                               | NO   | `now()`             |                                                       |
| `updatedAt`     | `timestamptz`                               | NO   | `now()`             |                                                       |
| `deletedAt`     | `timestamptz`                               | YES  | —                   |                                                       |

**Indexes:**

- UNIQUE `email` — one email, one user.
- Index `deletedAt` — for admin filtering.

**Check constraints:**

- `system_role IN ('user', 'admin')`
- `locale IN ('en', 'uk', 'ru')`
- `theme IN ('light', 'dark', 'system')`

**Admin promotion:** handled in the Auth.js `signIn` or `jwt` callback by comparing `email` against `process.env.ADMIN_EMAILS` (comma-separated). Must be idempotent (repeated logins yield consistent state) and reversible (removing email from env demotes on next login).

### `accounts`

Auth.js standard. OAuth provider link per user. One user can have multiple accounts (e.g., Google + future email/password).

| Column              | Type                     | Null | Notes                                            |
| ------------------- | ------------------------ | ---- | ------------------------------------------------ |
| `userId`            | `uuid`                   | NO   | FK → `users.id` ON DELETE CASCADE.               |
| `type`              | `text`                   | NO   | `'oauth' \| 'email' \| 'credentials'` (Auth.js). |
| `provider`          | `text`                   | NO   | `'google'` in MVP.                               |
| `providerAccountId` | `text`                   | NO   | User's ID at the provider.                       |
| `refreshToken`      | `text` (`refresh_token`) | YES  | OAuth refresh token.                             |
| `accessToken`       | `text` (`access_token`)  | YES  | OAuth access token.                              |
| `expiresAt`         | `bigint` (`expires_at`)  | YES  | Unix timestamp.                                  |
| `tokenType`         | `text` (`token_type`)    | YES  | `'Bearer'`.                                      |
| `scope`             | `text`                   | YES  | OAuth scopes.                                    |
| `idToken`           | `text` (`id_token`)      | YES  | OpenID token.                                    |
| `sessionState`      | `text` (`session_state`) | YES  | OpenID session state.                            |

**Primary key:** composite `(provider, providerAccountId)`.

**Indexes:** index on `userId`.

**FKs:** `userId → users.id ON DELETE CASCADE`.

**No extensions.** This is a pure Auth.js service table.

### `sessions`

Active sessions. Carries custom session policy fields.

| Column            | Type          | Null | Default | Notes                                             |
| ----------------- | ------------- | ---- | ------- | ------------------------------------------------- |
| `sessionToken`    | `text`        | NO   | —       | **Primary key.** Token from cookie.               |
| `userId`          | `uuid`        | NO   | —       | FK → `users.id` ON DELETE CASCADE.                |
| `expires`         | `timestamptz` | NO   | —       | Sliding expiration. Updated on activity.          |
| `rememberMe`      | `boolean`     | NO   | `false` | Extension. Flag for "Remember me".                |
| `absoluteExpires` | `timestamptz` | YES  | —       | Extension. Hard deadline for rememberMe sessions. |

**Indexes:**

- Index `userId` — for listing user's active sessions.
- Index `expires` — for periodic cleanup of expired sessions.

**No `createdAt`/`updatedAt`/`deletedAt`.** Sessions are service data and are hard-deleted on expiration.

**Session policy enforcement** (in Auth.js callbacks):

- Without "Remember me": `expires = now + 1h`, sliding update on activity.
- With "Remember me": `expires = min(now + 7d, absoluteExpires)`, `absoluteExpires = loginTime + 30d`.
- If `absoluteExpires < now` → session invalid.

### `verification_tokens`

Auth.js standard. Empty in MVP. Required by adapter.

| Column       | Type          | Null | Notes                        |
| ------------ | ------------- | ---- | ---------------------------- |
| `identifier` | `text`        | NO   | Usually email.               |
| `token`      | `text`        | NO   | One-time verification token. |
| `expires`    | `timestamptz` | NO   | Token expiration.            |

**Primary key:** composite `(identifier, token)`.

**Check constraints:**

- `valid_to IS NULL OR valid_to > valid_from`

**Exclusion constraint (`contracts_no_overlap_excl`):**

```sql
EXCLUDE USING gist (
  service_id WITH =,
  tstzrange(valid_from, valid_to, '[)') WITH &&
) WHERE (deleted_at IS NULL)
```

No two active contracts for the same service can overlap in time.

- `valid_to IS NULL OR valid_to > valid_from`
- `removed_at IS NULL OR installed_at IS NULL OR removed_at > installed_at`

**Exclusion constraint:** no two active meters for the same `(propertyId, serviceTypeId)` can overlap in time.

**Cross-table invariant (application-enforced):** if the related `service_type.supportsZones = false`, then `zoneCount` must be `1`.

### `readings`

Meter readings.

| Column      | Type            | Null | Default             | Notes                                   |
| ----------- | --------------- | ---- | ------------------- | --------------------------------------- |
| `id`        | `uuid`          | NO   | `gen_random_uuid()` | Branded `ReadingId`.                    |
| `meterId`   | `uuid`          | NO   | —                   | FK → `meters.id` ON DELETE CASCADE.     |
| `readAt`    | `timestamptz`   | NO   | —                   | When reading was taken (user-declared). |
| `valueT1`   | `numeric(12,3)` | NO   | —                   | Zone 1 / single-zone value.             |
| `valueT2`   | `numeric(12,3)` | YES  | —                   | Zone 2. NULL for single-zone meters.    |
| `valueT3`   | `numeric(12,3)` | YES  | —                   | Zone 3.                                 |
| `notes`     | `text`          | YES  | —                   |                                         |
| `createdBy` | `uuid`          | NO   | —                   | FK → `users.id` ON DELETE SET NULL.     |
| `createdAt` | `timestamptz`   | NO   | `now()`             |                                         |
| `updatedAt` | `timestamptz`   | NO   | `now()`             |                                         |
| `deletedAt` | `timestamptz`   | YES  | —                   |                                         |

**Indexes:**

- Index `(meterId, readAt)` — for time-ordered reading history.
- Index `createdBy`.
- Index `deletedAt`.

**Check constraints:**

- `value_t1 >= 0`
- `value_t2 IS NULL OR value_t2 >= 0`
- `value_t3 IS NULL OR value_t3 >= 0`

**Cross-table invariants (application-enforced):**

- `valueT2`/`valueT3` non-null iff `meter.zoneCount >= 2`/`= 3`.
- `readAt` must fall within `[meter.validFrom, meter.validTo ?? ∞)`.

**UX warning (not DB constraint):** reading values should typically monotonically increase. UI warns but permits decreases (legitimate cases: meter replacement, overflow rollover, input correction).

**No exclusion constraint on readings.** Multiple readings on the same date are permitted (rare but valid; UI guides users away from accidental duplicates).

- Index `(serviceId, periodStart, periodEnd)`.
- Index `createdBy`.

**Check constraints:**

- `amount >= 0`
- `period_end >= period_start`

## Application-Level Invariants

Constraints that are not expressible at the database level (require cross-table data, multi-row logic, or user-facing UX) and must be enforced in service code and covered by tests.

### Access control

- **Each property has at least one owner.** Removal or role downgrade of the last owner must fail.
- **Only owners can grant, modify, or revoke access.**
- **Owners cannot remove other owners.** Other owners must remove themselves.
- **Any user can always remove themselves** from a property access.

### Temporal data

- **Tariff, AccountNumber, PaymentDetails validity** must fall within the parent Contract's `[validFrom, validTo)`.
- **Meter/Reading consistency:** Reading's `readAt` must fall within the active window `[meter.validFrom, meter.validTo ?? ∞)` of its Meter.

### Service types

- **Tariff shape matches ServiceType's measurement type:**
  - `measurementType = 'metered'` → tariff uses `rateT1` (required), `rateT2`/`rateT3` optional per zone count. `fixedAmount` must be NULL.
  - `measurementType = 'fixed'` → tariff uses `fixedAmount` (required). All `rateT*` must be NULL.
- **Meter zone count consistent with ServiceType:** if `supportsZones = false`, `zoneCount` must be 1.
- **Reading zone values consistent with Meter:** `valueT2` non-null iff `zoneCount >= 2`; `valueT3` non-null iff `zoneCount = 3`.

### Deletion

- **Soft delete cascades via application transaction.** Deleting a property soft-deletes all related services, contracts, tariffs, account numbers, payment details, meters, readings, bills, payments.
- **Hard delete requires prior soft delete.** Admin-only operation. Records must have `deleted_at IS NOT NULL` before hard-delete is permitted.
- **Files in external storage (Google Drive, v2+)** are never touched by our delete operations.

### Provider integrity

- **Providers cannot be deleted if referenced by active contracts.** Both hard delete (via FK RESTRICT) and soft delete (via application check).

### Session policy

- **Admin promotion via `ADMIN_EMAILS` env** must be idempotent and reversible on each login.
- **Sliding expiration** updates `sessions.expires` on activity (throttled via `updateAge`).
- **Absolute expiration** for "Remember me" sessions is never extended past `absoluteExpires`.
